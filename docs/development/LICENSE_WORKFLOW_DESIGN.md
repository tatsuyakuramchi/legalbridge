# ライセンス契約導線設計

## 目的

ライセンス契約系は、`ライセンス基本契約` と `個別利用許諾条件` の役割を分ける。

- `ライセンス基本契約`
  - 当事者情報
  - 契約日
  - 一般条項
  - 基本的なライセンス対象情報
- `個別利用許諾条件`
  - 案件ごとの素材
  - 許諾範囲
  - 地域・言語
  - 金銭条件
  - 複数同時作成

この分離により、基本契約は長期利用、個別条件は案件単位で複数発行、という実務に合わせる。

## 推奨フロー

1. Slack で `ライセンス契約` を選択する
2. 最初に作成方式を選ぶ
   - `基本契約 + 個別利用許諾条件`
   - `個別利用許諾条件のみ`
3. Backlog に必要な課題を起票する
4. UI で内容を確認・編集する
5. 文書を生成する
6. 承認・押印へ進める
7. 後続で `利用許諾料計算` を行う

## Slack 導線案

### 1. 入口

`/法務依頼` で `ライセンス契約` を選択したとき、最初に以下を選ばせる。

- `基本契約 + 個別利用許諾条件`
- `個別利用許諾条件のみ`

### 2. ライセンス基本契約で聞く項目

基本的には `業務委託基本契約` に近い構成で扱う。

- 相手方名
- 契約書番号
- 契約日
- 相手方住所
- 相手方代表者
- 管轄裁判所
- 原著作物
- 原著作者
- クレジット表記
- 特記事項
- 備考

### 3. 個別利用許諾条件で聞く項目

個別条件はライセンス固有情報を集約する。

- 親ライセンス課題キー
- ライセンス種別名
- 原著作物
- 許諾開始日
- 地域
- 言語
- 素材番号
- 素材名
- 素材権利者
- 監修者
- 金銭条件1
- 金銭条件2
- 金銭条件3
- 特記事項
- 備考

### 4. 送信後ショートカット

- `ライセンス基本契約` 起票後
  - `個別利用許諾条件を作成`
- `個別利用許諾条件` 起票後
  - `利用許諾料計算を作成`

## Backlog 課題構成

### ライセンス基本契約

- 課題タイプ: `ライセンス契約`
- 役割: 基本契約の正本
- 主データ:
  - 契約当事者
  - 契約日
  - 一般条項
  - 原著作物
  - 原著作者
  - クレジット表記

### 個別利用許諾条件

- 課題タイプ: `個別利用許諾条件`
- 役割: 基本契約にぶら下がる案件別条件
- 主データ:
  - 親ライセンス課題キー
  - 許諾対象
  - 素材
  - 地域・言語
  - 金銭条件

### 運用ルール

- `基本契約 + 個別利用許諾条件` の場合
  - 2課題を作成する
- `個別利用許諾条件のみ` の場合
  - `個別利用許諾条件` のみ作成する
  - 既存の基本契約がある前提で運用する
- `個別利用許諾条件` は複数同時作成を許容する

## UI 画面構成案

### 契約編集画面

`/admin/workflow/contracts`

課題タイプに応じて画面を分ける。

#### ライセンス基本契約

- 基本情報
  - 契約書番号
  - 契約日
- 自社情報
- 相手方情報
- ライセンス基本情報
  - 原著作物
  - 原著作者
  - クレジット表記
- 相手方支払先情報
  - 銀行名
  - 支店名
  - 口座種別
  - 口座番号
  - 口座名義(カナ)
- インボイス制度関連
  - 適格請求書発行事業者
  - 登録番号
- 特記事項

`口座情報は表示する`

理由:
- 基本契約書テンプレートで相手方口座情報を使うため
- 業務委託基本契約と頭書き構成を合わせ、取引先登録フローの補完項目を再利用しやすくするため

補足:

- ライセンス基本契約の頭書きは `業務委託基本契約` と同じ考え方で扱う
- 当事者情報は `住所 / 商号 / 代表者` を基本とし、相手方側には `電話番号 / メール` も表示する
- 特約・特記事項は固定文を埋め込まず、UI / Backlog で保持した `REMARKS` をそのまま表示する

#### 個別利用許諾条件

- 基本情報
  - 親ライセンス課題キー
  - 契約書番号
- 当事者表示
- ライセンス個別条件
  - ライセンス種別名
  - 許諾開始日
  - 地域
  - 言語
- 素材情報
- 金銭条件1
- 金銭条件2
- 金銭条件3
- 特記事項

`口座情報は表示しない`

理由:
- 個別条件はライセンス支払条件の表示主体であり、契約書としての振込先表示は基本契約または DB 正本で管理する方が自然

## 実装方針

### 第1段

- Slack:
  - `ライセンス契約` 選択後に作成方式を選ばせる
- Backlog:
  - `ライセンス契約`
  - `個別利用許諾条件`
  の2課題構成を前提にする
- UI:
  - `ライセンス契約` と `個別利用許諾条件` で表示項目を完全に分ける

### 第2段

- `ライセンス契約` 起票後の Slack 返信に
  - `個別利用許諾条件を作成`
  ボタンを追加
- `個別利用許諾条件` 起票時に
  - 親ライセンス課題キーを自動セット

### 第3段

- `個別利用許諾条件` の複数同時作成
- 一覧から複数の個別条件をまとめて確認・生成
- 後続の `利用許諾料計算` との連携強化

## 今回の採用案

採用するのは次のルール。

- `ライセンス基本契約` の入力は `業務委託基本契約` に近づける
- `ライセンス情報` の中心は `個別利用許諾条件` に寄せる
- Slack では `基本契約 + 個別条件` または `個別条件のみ` を選べるようにする
- UI では `ライセンス契約` と `個別利用許諾条件` の編集画面を分ける
