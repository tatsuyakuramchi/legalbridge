# Operation Readiness Checklist

## 目的

現在の実装状態を前提に、ローカルUI、Backlog、Slack の順で確認するための実行チェックリスト。

## GCP 2サービス構成との関係

この資料はローカル一体運用の確認観点を残したチェックリストです。
`command-service / work-service` を前提にした GCP 運用へ移る場合は、次の資料を先に参照してください。

- `docs/development/GCP_TARGET_ARCHITECTURE.md`
- `docs/operations/GCP_SERVICE_SPLIT_RUNBOOK.md`
- `docs/operations/GCP_DEPLOY_SEQUENCE.md`
- `docs/operations/GCP_RETRY_AND_IDEMPOTENCY.md`

## サンプル通し確認の進め方

### 契約書生成

- [ ] `NDA / 業務委託基本契約 / ライセンス契約` のいずれか 1 件をサンプル課題として選ぶ
- [ ] `/admin/workflow/contracts` で課題キーを入れて `条件を確認` を押す
- [ ] `生成前チェック` の停止項目を解消する
- [ ] `文面プレビュー` でテンプレートと想定保存先を確認する
- [ ] `下書きを保存` してから `文書を生成` を押す
- [ ] `生成サマリー / 次に見ること / 生成済み文書` を確認する

### 納品帳票生成

- [ ] `親発注課題キー` と `今回納品金額` が入った `納品リクエスト` を 1 件用意する
- [ ] `/admin/workflow/delivery` で課題キーを入れて `条件を確認` を押す
- [ ] `親課題 / DeliveryEvent / 支払条件` の停止項目を解消する
- [ ] `出力見込み` で検収書のみか、支払通知書まで出るかを確認する
- [ ] `文書を生成` して `生成サマリー / 次に見ること` を確認する
- [ ] `生成サマリー` に `Backlog状態更新: 処理済み` が出る
- [ ] 一括検収書作成では `成功件数 / 要確認件数` が分かれて表示される
- [ ] 一括検収書作成で `要確認` になった課題は、課題リンクとエラー内容を追える

### 利用許諾料計算

- [ ] `ライセンス課題キー / 数量 / MSRP` が入った `製造案件` を 1 件用意する
- [ ] `/admin/workflow/royalty` で課題キーを入れて `条件を確認` を押す
- [ ] `ライセンス紐付け / 製造情報 / 計算条件` の停止項目を解消する
- [ ] `文書を生成` して `生成サマリー / 次に見ること` を確認する

## 1. ローカルUI

### 起動確認

- [ ] `SKIP_SLACK_STARTUP=1` でアプリを起動する
- [ ] `3000` が埋まっている場合でも `3100` にフォールバックして起動できる
- [ ] `/health` が `ok` を返す
- [ ] `/ready` が `ready: true` を返す
- [ ] `/status` で `DB / Slack / Backlog Config / Scheduler / Poller` の状態が見える
- [ ] `/status.json` で構造化ステータスを取得できる
- [ ] 管理UIの各画面が開く
  - [ ] `/admin/masters`
  - [ ] `/admin/settings/mapping`
  - [ ] `/admin/settings/workflow`
  - [ ] `/admin/orders/csv`
  - [ ] `/admin/workflow/stamp`

### マスタ管理

- [ ] Vendor サンプルCSVをダウンロードできる
- [ ] Staff サンプルCSVをダウンロードできる
- [ ] Vendor CSV が `UTF-8 / Shift_JIS` のどちらでも文字化けせずに読める
- [ ] Staff CSV が `UTF-8 / Shift_JIS` のどちらでも文字化けせずに読める
- [ ] Vendor CSV 一括登録が通る
- [ ] Staff CSV 一括登録が通る
- [ ] 一括登録後の完了メッセージに文字コードが表示される
- [ ] 文字化けデータを再取込した場合、同じ `slackUserId` の Staff が上書き更新される
- [ ] Vendor 個別登録が通る
- [ ] Staff 個別登録が通る

### 発注書系

- [ ] `.xlsx` をそのまま読み込める
- [ ] シート選択ができる
- [ ] 企画発注書マッピング設定が読める
- [ ] マッピング設定を保存できる
- [ ] プレビューに警告が出る
  - [ ] `vendorID` 未登録
  - [ ] 名前差分
  - [ ] 金額 0 円
  - [ ] 必須日付不足
- [ ] Vendor 仮登録導線が動く
- [ ] CSV / XLSX 取込から dry-run 相当の出力へ進める

### ワークフロー設定

- [ ] 部署未設定時の既定値として投稿チャンネル ID を保存できる
- [ ] 部署未設定時の既定値として承認（押印）ID を保存できる
- [ ] 部署未設定時の既定値として実行（押印）ID を保存できる
- [ ] Staff テーブル由来の部署候補が入力候補に出る
- [ ] 部署別設定を保存できる
- [ ] 部署別設定一覧を読み込める
- [ ] ルーティング確認で
  - [ ] 部署
  - [ ] 投稿チャンネル
  - [ ] 上長ID
  - [ ] 承認（押印）ID
  - [ ] 実行（押印）ID
  - [ ] 解決元
  が見える

### 押印管理

- [ ] 押印申請を送れる
- [ ] 方式更新ができる
- [ ] 押印完了登録ができる
- [ ] 差戻し登録ができる
- [ ] 一覧に結果が反映される

## 3. Slack

### App 設定

- [ ] App が Workspace にインストールされている
- [ ] Socket Mode が ON または Cloud Run の HTTP 受付へ切り替え済み
- [ ] Bot scopes が設定済み
  - [ ] `chat:write`
  - [ ] `commands`
  - [ ] `files:read`
  - [ ] `im:history`
- [ ] Event subscriptions に `message.im` がある
- [ ] Slash command がある
  - [ ] `/法務依頼`
  - [ ] `/法務検索`
  - [ ] `/法務ステータス`
  - [ ] `/法務一覧`
- [ ] Cloud Run 運用時は Slack App の Request URL が設定済み
  - [ ] Slash Command: `https://.../slack/commands`
  - [ ] Interactivity: `https://.../slack/interactions`

### 文書依頼フロー

- [ ] `/法務依頼` でモーダルが開く
- [ ] `/法務検索` で Backlog 案件検索ができる
- [ ] `/法務検索` の結果から後続申請モーダルへ進める
- [ ] 契約種別切替で入力欄が変わる
- [ ] 法務相談 では登録番号 / 相手方 / 住所が任意で起票できる
- [ ] NDA の必須チェックが効く
- [ ] 業務委託基本契約の必須チェックが効く
- [ ] ライセンス契約の必須チェックが効く
- [ ] 個別利用許諾条件の必須チェックが効く
- [ ] 売買契約3種の必須チェックが効く
- [ ] 納品リクエストの親課題キー必須が効く
- [ ] 利用許諾料計算の必須チェックが効く
- [ ] Cloud Run 受付では `DATABASE_URL` がなくても Backlog 起票まで継続する
- [ ] Cloud Run 受付では `SLACK_LEGAL_CHANNEL` がなくても受付自体は失敗しない

### 納期アラート

- [ ] 発注明細の納期 7 日前で案件スレッドに通知される
- [ ] 発注明細の納期 7 日前で申請者 + 上長メンション付き通知になる
- [ ] 発注明細の納期 3 日前で通知される
- [ ] 発注明細の納期当日に通知される
- [ ] 発注明細の納期超過後に毎日通知される
- [ ] `完了 / 破棄 / 締結済` の案件では通知されない
- [ ] 同じ日・同じ明細・同じタイミングで二重送信されない

### 課題キー通知と後続申請

- [ ] 部署設定の投稿チャンネルに親投稿が作成される
- [ ] 部署設定がない場合は既定の投稿チャンネルに親投稿が作成される
- [ ] 親投稿で申請者にメンションされる
- [ ] 部署設定に上長 Slack ID がある場合、親投稿で上長にメンションされる
- [ ] 受付後のアンサーバックが指定チャンネル側に集約される
- [ ] `発注書 / 企画発注書` で `納品リクエストを作成` が出る
- [ ] `ライセンス契約 / 個別利用許諾条件` で `利用許諾料計算を作成` が出る
- [ ] ショートカット押下で課題キー初期値が入る

### 承認・押印

- [ ] 部署別設定どおり承認依頼が飛ぶ
- [ ] 承認Slack通知に
  - [ ] 申請者
  - [ ] 部署
  - [ ] 解決元
  が出る
- [ ] 押印依頼が二重送信されない
- [ ] 押印依頼Slack通知に
  - [ ] 申請者
  - [ ] 部署
  - [ ] 解決元
  が出る
- [ ] 押印関連のアンサーバックが指定チャンネルに集約される
